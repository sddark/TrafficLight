The user is an engineer (likely embedded/systems) who:
- Works from formal system specification documents (with document IDs, version numbers, and numbered requirements like FR-xxx, NF-xxx).
- Expects architecture documents to be complete, implementation-ready, and leave zero design decisions to individual engineers.
- Uses a structured project layout: Docs/ directory contains both the system specification and architecture document.
- Works on Windows 11 with a Git-tracked project in a dedicated folder per project.
- Targets Raspberry Pi as the hardware platform with Python as the implementation language.
- Values precise numerical specs (units, tolerances, ranges) over vague language.
- Requests documents that a developer can implement from without needing to refer back to the conversation.
